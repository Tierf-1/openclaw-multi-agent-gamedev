## 一、产品愿景

OpenClaw 定位为 **"游戏开发领域的 Cursor/Devin"** —— 用户通过自然语言对话驱动 8 个 AI Agent 协作完成游戏开发全流程。

### 核心体验三要素
1. **对话即开发** (Chat-driven Development) — 聊天框是唯一入口
2. **过程可见** (Transparent Process) — Agent 工作过程实时可视化
3. **结果可交付** (Deliverable Output) — 每个阶段产出可下载的交付物

### 2.1 Dashboard 首页 (`/dashboard`) — P0

**当前状态**: ✅ 已完成 GPT/Grok 风格改造（聊天区 + 右侧项目历史栏）

**升级计划**:

| 功能点 | 描述 | 优先级 |
|--------|------|--------|
| 多轮对话支持 | 支持上下文连续对话，不只是单次输入 | P0 |
| 消息流展示 | 用户消息 + Agent 回复以聊天气泡形式展示 | P0 |
| 实时状态推送 | WebSocket 推送 Agent 工作状态和进度 | P0 |
| 快速操作菜单 | 输入框支持 `/` 命令（如 `/新游戏`、`/bug修复`） | P1 |
| 对话历史分组 | 右侧栏按"今天/本周/更早"分组显示 | P1 |
| 拖拽上传 | 支持拖拽图片/文件到输入框作为附件 | P2 |
| 对话收藏/置顶 | 重要对话可收藏或置顶 | P2 |

**交互设计要点**（UX小林）:
- 消息气泡样式: 用户消息靠右（品牌色背景），Agent 消息靠左（白色卡片）
- Agent 回复时显示"打字中..."动画 + Agent 头像
- 滚动到底部时自动跟随新消息
- 空状态引导: 首次访问展示欢迎语 + 示例需求

### 3.2 🔥 实时 Agent 工作状态可视化 — P0

**功能描述**: 用户可以实时看到每个 Agent 当前在做什么——不只是"进行中"状态点，而是具体的工作内容。

**展现形式**:
```
┌──────────────────────────────────────┐
│ 🔧 主程老陈 正在工作...               │
│ ┌────────────────────────────────┐   │
│ │ > 分析需求中...                 │   │
│ │ > 设计数据模型: Player, Enemy  │   │
│ │ > 生成技术方案文档...           │   │
│ │ █████████░░ 65%                │   │
│ └────────────────────────────────┘   │
│ 预计剩余: 2分钟                       │
└──────────────────────────────────────┘
```

**技术方案**:
- 后端: WebSocket 端点 `/ws/pipeline/{id}/status`
- Agent 执行过程中通过 WebSocket 推送进度事件
- 前端: 订阅 WebSocket，实时渲染进度面板

**负责人**: 主程老陈(03) 架构 → 程序小赵(04) 前后端实现

## 六、技术要点

### 6.1 WebSocket 通信协议

```typescript
// 前端订阅
const ws = new WebSocket(`ws://localhost:8080/ws/pipeline/${pipelineId}`)

// 消息类型
interface WSMessage {
  type: 'agent_status' | 'stage_update' | 'chat_reply' | 'deliverable_ready'
  agent_id: string
  data: {
    progress?: number
    message?: string
    status?: string
    file_url?: string
  }
  timestamp: string
}
```

### 6.2 对话历史数据模型

```typescript
interface Conversation {
  id: string
  title: string         // 自动从第一条消息提取
  created_at: string
  updated_at: string
  pipeline_id?: string  // 关联的项目
  messages: Message[]
}

interface Message {
  id: string
  role: 'user' | 'agent' | 'system'
  agent_id?: string     // 哪个Agent回复的
  content: string
  attachments?: Attachment[]
  timestamp: string
}
```

### 6.3 交付物管理

```python
# 后端API
@app.get("/api/pipelines/{pipeline_id}/deliverables")
@app.get("/api/deliverables/{deliverable_id}/download")
@app.post("/api/pipelines/{pipeline_id}/deliverables")
```
